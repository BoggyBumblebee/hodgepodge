Hodgepodge Help

Use Homebrew with a clearer, safer macOS interface.

Hodgepodge is a native macOS utility for browsing packages, understanding local Homebrew state, running common package and service actions, working with Brewfiles, and reviewing maintenance tasks without hiding the underlying brew commands.

The app is designed as a single main window with a separate Settings window. It is not a document app, and it manages the Homebrew installation on the current Mac.

Quick Start

  1. Open About Brew first to confirm Hodgepodge has detected your Homebrew installation.
  2. Open Catalog to browse formulae and casks from the hosted Homebrew API.
  3. Use Installed to inspect what is on this Mac and to perform most package actions.
  4. Use Outdated when you want to review or upgrade packages with newer versions available.
  5. Open Settings to choose launch, notification, Brewfile, and history behavior.

If you prefer keyboard access, the Help menu includes entries for this guide, Quick Start, and Troubleshooting.

Window Layout

Sidebar

The sidebar contains the main sections of the app. Choosing a section changes the main content area without opening a new document window.

Center List

Most sections use the center column as a searchable, sortable list. Selecting an item loads its details in the right-hand panel.

Detail Panel

The right-hand panel is where most actions happen. Hodgepodge keeps primary buttons near the top, with metadata, command previews, and output lower down.

  • Toolbar controls are section-specific and usually include search, refresh, filter, or sort actions.
  • Inline copy icons copy the exact underlying brew command where a command preview is shown.
  • Command output is hidden by default and can be expanded when you need technical detail.

About Brew

About Brew is the installation overview. Use it to confirm that Hodgepodge has located the Homebrew executable, repository, prefix, cellar, version, and related installation metadata.

  • Use Refresh Homebrew if the installation details look stale.
  • Use the help buttons there as a quick route into this bundled guide.
  • If Hodgepodge cannot locate Homebrew, start troubleshooting here before using the other sections.

Catalog

Catalog is the hosted package browser. It shows formulae and casks from the public Homebrew API, not just what is installed on this Mac.

What it is for

  • Discovering packages
  • Reviewing package detail and metadata
  • Installing or fetching packages
  • Favoriting packages and saving searches

What to look for

  • Package name, type, and install state
  • Homepage and download icons in the header
  • Primary action button that changes based on local state
  • Command history for package actions

If a package is already installed, the primary button changes from Install to Uninstall. Catalog also stays in sync with local state updates from other sections.

Catalog Analytics

Catalog Analytics shows leaderboards based on Homebrew's public analytics data, including top installs and common build errors.

  • Use the period control to switch between the available reporting windows.
  • Installed items are marked and can take you directly to Installed.
  • Non-installed items take you to Catalog and select the relevant package.

Installed

Installed is the main local inventory view. It reads your installed formulae and casks from Homebrew's local JSON output and presents package state, dependency information, and most package actions.

Use it for

  • Reviewing installed formulae and casks
  • Filtering by package state, including favorites
  • Exploring dependency trees and dependents
  • Uninstall, reinstall, link, unlink, pin, and unpin

Brewfile support

You can generate a Brewfile from the installed inventory here. The default export scope is controlled by Settings.

When package state changes here, related sections such as Catalog, Outdated, and Services refresh to stay in sync.

Outdated

Outdated shows installed packages that have a newer version available. Use it to review upgrade opportunities before running them.

  • Use Upgrade Now for a single package.
  • Use Upgrade All to act on the currently visible upgradeable list.
  • Pinned packages are shown clearly and cannot be upgraded until unpinned.

Services

Services is for Homebrew-managed background services. It combines service list and detail data so you can see status and run service actions from one place.

  • Start, stop, restart, and kill individual services.
  • Run service cleanup when needed.
  • Use the list to review status, user, and other service metadata.

Taps

Taps lists installed Homebrew taps and shows tap metadata, paths, remotes, and package counts.

  • Search or sort the currently installed taps.
  • Add a new tap from the toolbar flow.
  • Untap an existing tap from the detail header.
  • Use Force untap only when you understand the implications.

Brewfile

Brewfile is the bundle-management section. It can open an existing Brewfile, inspect its entries, validate it, install from it, export one, and add or remove supported entries.

Main tasks

  • Choose or reopen a Brewfile
  • Run brew bundle check
  • Run brew bundle install
  • Export or dump a Brewfile

Good to know

  • The selected Brewfile can be restored on launch if that preference is enabled.
  • Entry add and remove actions use Homebrew's own bundle commands rather than hand-editing the file.
  • Unsupported bundle entry kinds remain visible, even when no add action exists for them.

Maintenance

Maintenance centralizes common upkeep commands so you can review the state of Homebrew and run maintenance tasks without dropping straight to Terminal first.

  • Review configuration and doctor output.
  • Inspect cleanup and autoremove dry runs.
  • Run update, cleanup, autoremove, doctor, and config actions.

Settings

Hodgepodge uses a dedicated macOS Settings window for app-wide preferences. Open it from the standard app menu just like other native macOS applications.

General

  • Choose the default launch section.
  • Control whether the last selected Brewfile is restored.

Notifications

  • Enable or disable completion notifications.
  • Choose whether to notify for all actions or only long-running actions.
  • Choose which categories can raise notifications.

Brewfile and History

  • Set the default Brewfile export scope for the Installed section.
  • Set Catalog command-history retention.

Per-screen state such as search text, filters, selection, and temporary command-output expansion is kept out of Settings on purpose.

Commands and Safety

Hodgepodge aims to stay transparent to the underlying Homebrew CLI. Where actions are available, the app can show you the exact command it intends to run.

  • Primary actions are placed near the top of the detail area for the selected item.
  • Potentially destructive actions are typically confirmed before execution.
  • Raw command output is available, but hidden by default to keep the UI calmer.
  • Command history is stored for Catalog actions and can be reviewed later.

Tips

  • Use favorites and saved searches in Catalog to keep frequently used packages close.
  • Use Installed when you need package state and dependency context, not just package metadata.
  • Use Outdated when you want a cleaner upgrade-focused list than the full Installed inventory.
  • If a command result looks too minimal, expand the command details area to inspect the underlying output.
  • Use About Brew first when something feels generally wrong with the app's Homebrew view.

Troubleshooting

Homebrew is not detected

  • Confirm Homebrew is installed on this Mac.
  • Confirm brew exists at a standard location such as /opt/homebrew/bin/brew or /usr/local/bin/brew.
  • Return to About Brew and use Refresh Homebrew.

A section looks stale

  • Use the section refresh control or Command-R.
  • If you just changed package state elsewhere, reopen the section or trigger another refresh.
  • Make sure the underlying Homebrew command actually completed successfully.

Help does not open

  • Confirm the app bundle contains Help/index.html and Help/help.css.
  • Rebuild the app if the resources appear to be missing.

If a command succeeds in Terminal but fails in Hodgepodge, inspect the command details in the app and compare the exact command and output with your terminal run.
